## 备份策略

### 每日备份
- 使用 `Scripts/Maintenance/backup_and_cleanup.sh` 脚本进行每日备份
- 备份位置: `Backups/Daily/YYYYMMDD/`
- 备份内容: 过去24小时内修改的所有文件
- 备份报告: 桌面上生成的备份报告文件

### 特殊备份（发布前）
- 在每次发布前执行完整备份
- 使用 `Scripts/Maintenance/backup_release.sh` 脚本（如有）
- 备份位置: `Backups/Release/X.Y.Z/`
- 备份内容: 所有源代码和资源文件

### 特殊备份（重大更改前）
- 在进行重大架构更改前执行备份
- 手动备份或使用专门脚本
- 备份位置: `Backups/Special/[描述性名称]/`
- 备份内容: 与更改相关的文件

## 备份恢复指南

如需恢复备份，请按照以下步骤操作：

1. 确定需要恢复的备份日期和位置
2. 使用备份索引文件 (`backup_index.md`) 查找需要恢复的文件
3. 复制文件到项目相应位置
4. 如需恢复整个项目，建议创建新的工作目录并复制备份内容

或者使用恢复脚本（如有）：
```bash
./Scripts/Maintenance/restore_backup.sh [备份日期] [可选:文件路径]
```
