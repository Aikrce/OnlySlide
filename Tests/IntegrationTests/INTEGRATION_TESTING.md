# 集成测试

## 概述

这个目录包含 OnlySlide 的集成测试用例，用于验证不同模块之间的协作和交互。

## 测试范围

- 模块集成测试：验证模块间接口的兼容性
- 数据流测试：测试数据在系统中的完整流转
- 端到端测试：验证关键用户场景的完整流程
- 系统边界测试：测试与外部系统的集成

## 运行测试

执行以下命令运行所有集成测试：

```bash
swift test --filter IntegrationTests
```

要运行特定集成测试：

```bash
swift test --filter IntegrationTests.DataFlowTests
```

## 环境设置

集成测试需要特定的环境设置，详情请参考 `TestEnvironmentSetup.md` 文件。集成测试使用专门的测试数据库和模拟服务，以确保测试的独立性和可重复性。

## 添加新测试

添加新的集成测试时，请遵循以下规则：

1. 测试应该关注模块间交互，而非单个模块的内部实现
2. 使用合适的夹具(fixtures)准备测试数据
3. 合理划分测试边界
4. 记录测试涉及的组件和交互点

## 故障排查

如果集成测试失败，请参考 `IntegrationTestTroubleshooting.md` 文件了解常见问题和解决方案。
