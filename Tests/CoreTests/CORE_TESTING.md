# 核心模块测试

## 概述

这个目录包含 OnlySlide 核心模块的测试用例，验证核心业务逻辑和功能。

## 测试范围

- 业务逻辑测试：验证核心业务规则和流程
- 服务层测试：测试各种服务组件的功能
- 模型测试：确保数据模型的完整性和有效性
- 工具类测试：验证辅助工具和实用函数

## 运行测试

执行以下命令运行所有核心模块测试：

```bash
swift test --filter CoreTests
```

要运行特定测试类：

```bash
swift test --filter CoreTests.DataServiceTests
```

## 测试策略

核心模块测试采用单元测试和集成测试相结合的方式：

- 单元测试：使用模拟对象验证独立组件
- 集成测试：验证组件之间的交互

## 添加新测试

添加新测试时，请遵循以下规则：

1. 测试类使用 `XCTestCase` 子类，并以 `Tests` 结尾
2. 测试方法应该关注一个特定功能或场景
3. 使用合适的模拟对象隔离被测组件
4. 使用有描述性的断言消息，便于调试失败的测试
