# UI 测试

## 概述

这个目录包含 OnlySlide 的 UI 测试用例，用于验证用户界面的正确性和用户交互。

## 测试范围

- 界面元素测试：验证界面元素的正确显示和布局
- 用户交互测试：测试按钮点击、滑动等交互操作
- 导航测试：验证应用内导航和页面跳转
- 可访问性测试：确保应用符合可访问性标准
- 本地化测试：验证界面在不同语言环境下的表现

## 运行测试

执行以下命令运行所有 UI 测试：

```bash
swift test --filter UITests
```

要在模拟器上运行 UI 测试：

```bash
xcodebuild test -scheme OnlySlide -destination 'platform=iOS Simulator,name=iPhone 14,OS=16.0'
```

## 截图比较

UI 测试使用截图比较技术验证界面外观。参考截图存储在 `ReferenceImages` 目录中，测试生成的截图会与参考截图进行比较，以发现界面变化。

## 添加新测试

添加新的 UI 测试时，请遵循以下规则：

1. 使用 XCUITest 框架与界面交互
2. 创建清晰的测试场景，模拟真实用户行为
3. 添加适当的等待逻辑，处理界面加载和动画
4. 维护参考截图，确保它们反映预期的界面外观

## 故障调试

UI 测试失败时，可以查看 `test_artifacts` 目录中的截图和日志，帮助诊断问题。
