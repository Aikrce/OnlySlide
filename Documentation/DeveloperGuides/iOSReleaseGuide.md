# OnlySlide iOS发布指南

## 概述

本文档提供了OnlySlide应用从开发到在App Store发布的完整流程指南。按照本指南操作，可以确保应用程序顺利通过苹果审核并面向用户发布。

## 准备工作

### 1. 版本规划

在开始发布过程前，请确保完成以下规划工作：

- 确定要发布的功能集和修复的Bug
- 遵循[语义化版本控制](https://semver.org/)准则更新版本号
  - 主版本号（X.0.0）：不兼容的API变更
  - 次版本号（0.X.0）：向后兼容的功能性新增
  - 修订号（0.0.X）：向后兼容的问题修正
- 准备版本发布说明
- 检查所有功能是否按计划完成并通过测试

### 2. 代码冻结

1. 创建发布分支：
```bash
git checkout -b release/v1.x.x develop
```

2. 锁定依赖版本：
   - 确保Podfile或SPM依赖中使用明确的版本号，而非范围或分支
   - 对于自有依赖库，切换到稳定标签版本

3. 执行最终代码评审，关注：
   - 潜在的性能问题
   - 内存泄漏
   - 未使用的代码
   - 硬编码的测试数据或密钥

## 应用配置

### 1. 版本与构建号

更新以下文件中的版本信息：

1. 打开`Info.plist`文件并更新：
   - CFBundleShortVersionString（版本号，如"1.2.0"）
   - CFBundleVersion（构建号，如"1"，每次提交给App Store Connect时递增）

2. 为内部跟踪添加构建时间戳：
```swift
// 在AppDelegate或适当位置添加
let buildTimestamp = Date()
print("App build timestamp: \(buildTimestamp)")
```

### 2. 应用图标与启动屏幕

1. 确认所有必须的应用图标尺寸均已包含：
   - iOS设备：20pt, 29pt, 40pt, 60pt等（@1x, @2x, @3x）
   - iPad设备：20pt, 29pt, 40pt, 76pt, 83.5pt等（@1x, @2x）

2. 检查启动屏幕：
   - 确保显示正确的品牌标识
   - 适配所有支持的设备尺寸
   - 避免闪烁或不必要的过渡效果

### 3. 应用元数据

准备以下App Store Connect所需内容：

1. 应用描述（4000字符以内）
2. 关键词（100字符以内）
3. 支持网址与隐私政策URL
4. 联系信息（电子邮件、电话）
5. 版本发布说明（4000字符以内）
6. 营销材料
   - 应用截图（不同设备尺寸）
   - App Store预览视频（可选）

## 构建与测试

### 1. 最终测试

1. 执行完整的回归测试，确保：
   - 所有核心功能正常运行
   - UI/UX符合设计规范
   - 设备兼容性测试（iPhone、iPad等不同型号）
   - 系统版本兼容性（支持的最低iOS版本）

2. 执行性能测试：
   - 启动时间
   - 内存使用
   - 网络请求效率
   - 电池消耗

3. 边缘情况测试：
   - 低存储空间
   - 弱网络或离线状态
   - 系统中断（电话、通知等）

### 2. 生成发布构建

1. 选择正确的构建配置：
   - 在Xcode中，确保选择"Release"配置
   - 禁用所有调试功能和测试代码

2. 生成归档文件：
   - 在Xcode中，选择"Product > Archive"
   - 等待构建完成并打开"Organizer"窗口

3. 验证归档：
   - 在Organizer中，选择最新归档
   - 点击"Validate App"按钮
   - 解决任何验证警告或错误

## 提交到App Store

### 1. 上传构建

1. 从Xcode Organizer上传：
   - 选择验证通过的归档
   - 点击"Distribute App"
   - 选择"App Store Connect"
   - 按提示操作完成上传

2. 或使用Transporter工具上传：
   - 从Organizer导出.ipa文件
   - 打开Apple的Transporter应用
   - 上传.ipa文件

3. 等待处理：
   - 上传后，构建需要几分钟到几小时进行处理
   - 通过电子邮件或App Store Connect接收处理完成通知

### 2. 配置App Store Connect

1. 登录[App Store Connect](https://appstoreconnect.apple.com/)

2. 选择您的应用，并进入"iOS App"部分

3. 填写或更新所有必需信息：
   - 定价与可用性
   - 年龄分级
   - App Store信息（描述、关键词、截图等）
   - 版本发布说明

4. 选择发布选项：
   - 手动发布：审核通过后手动发布
   - 自动发布：审核通过后自动发布
   - 计划发布：指定发布日期和时间

5. 提交审核：
   - 检查所有必填字段
   - 回答审核问题（如使用特定iOS功能、敏感内容等）
   - 点击"Submit for Review"按钮

## 发布后管理

### 1. 监控发布状态

定期检查：
- App Store Connect中的应用状态
- 用户反馈与评论
- 崩溃报告与性能指标

### 2. 准备快速修复

对于发布后发现的严重问题：
1. 快速修复关键Bug
2. 提交审核时请求加急审核（适用于严重Bug）

### 3. 规划下一个版本

基于用户反馈和项目路线图，开始规划下一版本：
- 分析用户行为数据
- 优先处理常见用户请求
- 跟踪竞争对手更新

## 发布清单

使用以下清单确保所有重要步骤已完成：

- [ ] 版本号和构建号已更新
- [ ] 所有功能开发完成并通过测试
- [ ] 删除测试代码和调试标志
- [ ] 应用图标和启动屏幕已优化
- [ ] 配置切换到生产环境
- [ ] 执行全面回归测试
- [ ] 生成并验证发布归档
- [ ] 准备营销材料（截图、描述等）
- [ ] 应用程序元数据已更新
- [ ] 构建已上传并处理成功
- [ ] 提交审核申请

## 特殊情况处理

### 1. 审核被拒

如果应用被拒：
1. 仔细阅读拒绝原因
2. 根据反馈修复问题
3. 如果认为被错误拒绝，准备申诉材料
4. 修复后重新提交或提交申诉

### 2. 紧急更新

对于需要紧急发布的修复：
1. 在发布说明中清晰说明紧急修复的问题
2. 联系App Review团队请求加急审核
3. 在App Store Connect的"提交审核"页面选择"加急审核"选项并说明原因

## 结论

遵循此发布指南可以简化OnlySlide应用的发布流程，减少审核拒绝的可能性，并确保高质量的用户体验。对于每个新版本，请更新本文档以反映苹果政策变更或内部流程改进。
