# OnlySlide项目修复计划

## 问题总结

在分析项目后，我发现以下关键问题：

### 1. Core模块编译错误

经过代码检查，发现以下具体问题：

1. **类型引用错误**：`CoreDataMigrationManager`类引用了`AppError`，但使用方式与实际定义不符
2. **缺少实现**：在迁移相关代码中存在方法引用不一致的问题
3. **重复定义**：`CoreDataError`和`AppError`在多个位置有不同的定义

### 2. 目录结构混乱

1. **测试环境分离**：创建了单独的`CoreDataTests`目录和包，而非使用标准的Swift Package Manager测试结构
2. **重名文件**：存在多个同名的类和文件，如`CoreDataTests`、`CoreDataError`等
3. **符号链接问题**：在`Tests/CoreTests/Data`目录下存在指向`../CoreTests`的符号链接，形成循环引用

### 3. 其他问题

1. **依赖管理不一致**：主项目和独立测试包使用不同的依赖管理方式
2. **模型定义不同步**：主项目和测试环境使用不同的数据模型定义
3. **文档不足**：缺少清晰的架构和测试文档

## 解决方案

### 1. 修复Core模块编译错误

1. **统一错误定义**：
   - 将`AppError`和`CoreDataError`的定义统一
   - 确保引用正确的错误类型

2. **修复迁移管理器**：
   - 确保`CoreDataMigrationManager`和`CoreDataModelVersionManager`之间的接口一致
   - 修复方法调用不匹配的问题

### 2. 标准化目录结构

1. **整合测试框架**：
   - 将`CoreDataTests`合并到标准的`Tests`目录中
   - 使用Swift Package Manager的标准测试结构

2. **消除命名冲突**：
   - 明确重命名冲突的类和文件
   - 确保每个组件有明确的命名约定

3. **移除循环引用**：
   - 删除导致问题的符号链接
   - 使用更合理的引用方式

### 3. 完善项目文档

1. **添加架构文档**：
   - 清晰描述项目架构和模块关系
   - 说明设计决策和约定

2. **添加测试指南**：
   - 描述如何编写和运行测试
   - 解释测试架构和策略

## 实施计划

1. 首先修复Core模块编译错误
2. 标准化目录结构
3. 修复测试环境
4. 编写文档

## 预期成果

1. Core模块编译通过
2. 测试框架集成到标准结构中
3. 目录结构清晰，没有命名冲突
4. 完整的文档
